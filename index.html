Automation Overview

Missed-call flow designed to map cleanly into real webhook automation later.
Inbound event hits a webhook

Phone call, text message, or website form gets normalized into a single lead event format.

Lead Board creates or updates the lead

Each new event either creates a fresh lead or attaches to an existing contact record and timeline.

Auto-reply and task fire

Missed calls trigger the friendly text. Texts trigger a follow-up task. Everything lands in the lead detail view.

Core Data Model

Lead

id, createdAt, source, customerName, phone, email, messagePreview, fullMessage, status, assignedTo, lastActivityAt, quotedAmount, notes

LeadEvent

id, leadId, type, timestamp, payload for call, SMS, form, task, and status timeline entries.

Task + AutomationRule

Follow-up tasks attach to a lead and automation rules define reply templates and follow-up timing.

Real MVP Path

Recommended vendor target

Twilio for phone/SMS, webhook-backed backend service, and a database like Postgres or Supabase.

Operational next steps

Add authentication, delivery logging, SMS opt-out handling, assignment rules, and persistent lead ownership.

Deployment shape

Frontend deploys to Vercel or Netlify. Backend lives separately and processes all webhook events and automations.
